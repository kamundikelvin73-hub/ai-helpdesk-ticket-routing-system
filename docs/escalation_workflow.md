# Escalation Workflow

## High Priority

Escalate immediately when:

- User cannot access critical systems
- Security risk is reported
- Many users are affected
- Business operations are blocked

## Medium Priority

Handle within normal support workflow when:

- Single-user software issue
- Device performance issue
- Non-critical application problem

## Low Priority

Handle when:

- General questions
- Minor requests
- Documentation requests

```text

Impact:
Users affected:
Steps already taken:

```
